# ICP — SLAIVO CARGO OS

## 1. Target Customer (V1)

SLAIVO targets cargo / import-export agencies that:

- operate mainly via WhatsApp
- receive daily client messages
- manage shipments manually
- track payments manually
- rely on memory or Excel
- have one main decision-maker (manager)

## 2. Geography

Primary focus:

- Africa (DRC, Ivory Coast, Ghana, Mali, Burkina Faso, etc.)

But system must be:

- international-ready
- multi-country
- multi-currency

## 3. Operational Reality

These agencies:

- receive many repetitive questions
- lose track of client requests
- forget to follow up
- manage payments manually
- handle partial information
- deal with clients physically at the office
- manage supplier drop-offs abroad

## 4. Core Problems

- WhatsApp chaos
- lost leads
- missing follow-ups
- unclear shipment status
- unpaid balances
- no proof of actions
- mental overload for the manager

## 5. Why They Will Pay

They will pay because SLAIVO:

- reduces repetitive messages
- structures operations
- improves tracking
- increases payment recovery
- centralizes information
- reduces mistakes
- gives visibility

## 6. What They Are NOT

V1 is NOT for:

- large enterprises with ERP
- companies with full IT systems
- non-logistics businesses
- companies that do not use WhatsApp

## 7. Wedge Strategy

Start narrow:

WhatsApp-first cargo agencies

Then expand later to:

- TikTok acquisition
- Gmail documentation
- multi-channel
- multi-agency SaaS
